# Google Business Profile — Agency Setup & API Access

## Overview

TracPost applies as a GBP Agency to centrally manage business listings for all subscribers. This gives:
1. **Agency Dashboard** — Google's native UI for managing multiple locations
2. **Business Profile API** — programmatic CRUD for listings, posts, reviews, insights
3. **Bulk verification** — streamlined verification for agencies managing 10+ locations

## What the API Can Do (TracPost Dashboard Replacement)

### Listing Management (fully automatable)
- `POST accounts.locations.create` — create new business listing
- `PATCH accounts.locations.patch` — update name, address, phone, hours, categories, description, website, attributes
- `DELETE accounts.locations.delete` — remove listing
- `GET accounts.locations.list` — list all managed locations

### Content Publishing (fully automatable)
- `POST accounts.locations.localPosts.create` — create posts (updates, offers, events, products)
- `PATCH accounts.locations.localPosts.patch` — edit posts
- `GET accounts.locations.localPosts.list` — list all posts
- Supports: text, photos, CTAs (call, book, learn more, order, sign up)

### Reviews (fully automatable)
- `GET accounts.locations.reviews.list` — fetch all reviews
- `PUT accounts.locations.reviews.updateReply` — respond to reviews
- `DELETE accounts.locations.reviews.deleteReply` — remove a reply
- Review notifications via webhook/polling

### Photos & Media (fully automatable)
- `POST accounts.locations.media.create` — upload photos
- `GET accounts.locations.media.list` — list media
- Categories: COVER, PROFILE, LOGO, EXTERIOR, INTERIOR, PRODUCT, AT_WORK, FOOD_AND_DRINK, MENU, COMMON_AREA, ROOMS, TEAMS, ADDITIONAL

### Insights & Analytics (fully automatable)
- `GET accounts.locations.reportInsights` — views, searches, actions, direction requests, phone calls, photo views
- Breakdown by time period, comparison periods
- Search query data (what people searched to find the business)

### Verification (semi-automatable)
- `POST accounts.locations.verifications.create` — initiate verification
- `POST accounts.locations.verifications.complete` — submit verification code
- Methods: POSTCARD, PHONE_CALL, EMAIL, AUTO (Google decides)
- **Limitation**: Human must receive and enter the code. API can initiate and submit, but not intercept.

### Q&A (fully automatable)
- Read and answer questions posted on the listing
- Post questions proactively (FAQ strategy)

## Agency Application Steps

### Step 1: Google Cloud Project
1. Go to `console.cloud.google.com`
2. Create project: **TracPost**
3. Enable APIs:
   - **Business Profile API** (`mybusinessbusinessinformation.googleapis.com`)
   - **My Business Account Management API** (`mybusinessaccountmanagement.googleapis.com`)
   - **My Business Verifications API** (`mybusinessverifications.googleapis.com`)
   - **My Business Lodging API** (optional)
   - **My Business Q&A API** (optional)
   - **My Business Notifications API** (optional)
4. Create OAuth 2.0 credentials:
   - Application type: Web application
   - Authorized redirect URI: `https://tracpost.com/api/auth/google/callback`
   - Save Client ID + Client Secret
5. Add to env vars: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`

### Step 2: Agency Registration
1. Go to `business.google.com/agencysignup` (or through Google's Agency program)
2. Sign in with the TracPost Google account
3. Fill in agency details:
   - **Agency name**: TracPost (by Eppux LLC)
   - **Website**: `https://tracpost.com`
   - **Agency type**: Digital marketing / Social media management
   - **Number of locations managed**: Estimate (start with 10-50)
4. Accept agency terms
5. Link the Google Cloud project

### Step 3: Organization Account
1. In Business Profile Manager (`business.google.com`), create an Organization
2. Organization name: **TracPost**
3. This becomes the parent container for all subscriber locations
4. Invite the TM Google account as a manager

### Step 4: OAuth Scopes
Request these scopes in your OAuth consent screen:
- `https://www.googleapis.com/auth/business.manage` — full management access
- `https://www.googleapis.com/auth/userinfo.email` — user identification

### Step 5: OAuth Consent Screen
In Google Cloud Console → APIs & Services → OAuth consent screen:
- **App name**: TracPost
- **User support email**: support@tracpost.com
- **App logo**: TracPost logo
- **App domain**: tracpost.com
- **Privacy policy**: https://tracpost.com/privacy
- **Terms of service**: https://tracpost.com/terms
- **Authorized domains**: tracpost.com
- **Developer contact**: admin@tracpost.com

### Step 6: Verification (if publishing to external users)
- Submit for Google OAuth verification review
- Required for apps accessing sensitive scopes (business.manage)
- Timeline: 1-4 weeks
- Requires privacy policy, terms, and homepage

## Provisioning Workflow with API

### Per-Subscriber (automated from provisioning page):
```
1. Create location via API:
   POST /v1/accounts/{agencyAccountId}/locations
   {
     "locationName": "Epicurious Kitchens",
     "primaryCategory": { "categoryId": "gcid:kitchen_remodeler" },
     "address": { "locality": "Pittsburgh", "administrativeArea": "PA", ... },
     "primaryPhone": "+14125551234",
     "websiteUri": "https://epicuriouskitchens.com",
     "profile": { "description": "Bio from Profile Kit" }
   }

2. Initiate verification:
   POST /v1/accounts/{agencyAccountId}/locations/{locationId}/verifications
   { "method": "PHONE_CALL" }  // or POSTCARD, EMAIL

3. Store location ID in TracPost:
   → gbp_locations table (site_id, gbp_location_id, sync_status)

4. Once verified:
   → Pipeline publishes GBP posts alongside social content
   → Reviews sync to inbox
   → Insights feed analytics
```

### Admin handles verification manually:
- Receives postcard/phone code
- Enters via API: `POST /verifications/{id}/complete { "pin": "12345" }`
- Or enters via Google's dashboard (fallback)

## YouTube Channel (Same Google Cloud Project)

The same Google Cloud project + OAuth credentials work for YouTube:
- Enable **YouTube Data API v3** (`youtube.googleapis.com`)
- Add scope: `https://www.googleapis.com/auth/youtube.upload`
- OAuth callback already handles YouTube channel discovery

**One Google Cloud project → GBP + YouTube + Google account sign-in**
